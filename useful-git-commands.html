Git Commands

1. git diff

We had already staged our files initially above and made our initial commit.

Now suppose I modify a file say ‚ÄúTest.txt‚Äù and save it.

Let‚Äôs run the ‚Äúgit status‚Äù command.

As you can see it shows that we have modified that file. Now let‚Äôs run the ‚Äúgit diff ‚Äù command and see what happens.

Here using the ‚Äúgit diff ‚Äù command we are actually comparing this file ‚ÄúTest.txt‚Äù(modified file) which is in our current working directory with the same file which we had staged earlier(i.e in the Staging Area).

So using this command we can come to know what modifications did we do in any file in our current Working Directory with the same file in Staging Area.

Here as you can see the ‚Äò+‚Äô symbol denotes the changes that we made in that file(in our Working Directory) and ‚Äò-‚Äô denotes the content in the file in the Staging Area

2. git diff ‚Äî staged

This command is used to know the difference between the files in the previous commit and the current Staging Area.

Now let‚Äôs stage these files and run the ‚Äúgit diff ‚Äî staged ‚Äù command.

3. Renaming and Deleting any file using Git Bash

Deleting/Renaming files can be a bit confusing and sometimes tedious especially if your Git Repository has a lot of files and folders because you would have to then stage all the files as wellüò•.

A cleaner and efficient way üòäto do this would be to use the following Git commands:-

Deleting a file

Using these commands we can easily Rename/Delete any file in our Repository and the changes would also be staged automatically by Git so you wouldn‚Äôt have to worry about staging these changesüòç.

4. Syncing Forks in Git and Github

The World of Open Source Development is really active because so many developers contribute to different Open Source Projects.

So if you are contributing to an Open Source Project then Syncing your forked Repository with the Original Repository becomes really important.

Suppose you already have a Forked Repository(on Github) which you want to sync with the Original Repository(from which you forked).

If you haven‚Äôt cloned the Forked Repo on your Local machine then just use the following command:- 

git clone https://github.com/<Username>/<Repository-name>.git <filename>
# filename: It is the name of file/folder for the Cloned Repository on your Local Machine.

Now we will add the Original repository as a remote repository(upstream):- 

git remote add upstream https://github.com/<Owner>/<Repository>.git
# add the original repository as remote repository called "upstream"

Now we will fetch the changes from the upstream repository:- 

git fetch upstream
# Fetch the branches and their respective commits from the upstream repository.

We will now switch to the master branch of the fork‚Äôs local repo and merge the changes in the upstream repo to the local repo‚Äôs master branch:- 

git checkout master
# switch to the master branch of your fork's local repo

git merge upstream/master
# Merge the changes from upstream/master into your local master branch.

This brings your fork‚Äôs local repo in sync with the Original repositoryüòÑ.

However, if you also want to update the forked repo on Github then you would have to push the master branch (or the branch in which you merged the changes of the upstream repo). 

git push origin master
# origin: forked repository on Github

And boom!! Your Forked Repo on Github is also now synced with the Original RepoüòÑ.
